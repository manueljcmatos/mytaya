  <action>
    1. Create `telegram.ts` with raw Telegram Bot HTTP API functions (no SDK):

       Constants: `const TELEGRAM_API = 'https://api.telegram.org/bot';`

       `sendPhoto(token, channelId, photoUrl, caption): Promise<number | null>` - POST to sendPhoto endpoint with chat_id (channel @username or numeric ID), photo URL (R2 public URL), HTML-formatted caption, parse_mode:'HTML'. Returns message_id on success, null on failure. Log errors but do not throw.

       `sendMessage(token, channelId, text): Promise<number | null>` - POST to sendMessage endpoint for text-only messages (daily recap). parse_mode:'HTML'. Returns message_id.

       `buildPredictionCaption(pred)` - Bilingual caption format:
       ```
       {homeTeam} vs {awayTeam}
       Pick: {pickLabelEn} @ {odds}
       Hula: {pickLabelTl} @ {odds}

       https://mytaya.com/en/predictions/{slug}/
       ```

       `buildResultCaption(pred)` - Bilingual result caption:
       ```
       {homeTeam} {homeScore} - {awayScore} {awayTeam}
       Result: {WIN/LOSS/PUSH}
       Resulta: {PANALO/TALO/PUSH}

       https://mytaya.com/en/predictions/{slug}/
       ```

       `sendPredictionToTelegram(env, pred): Promise<void>` - Sends photo with prediction caption. Updates `published_telegram = true` and `telegram_message_id` in predictions table. Requires pred to have card_image_url.

       `sendResultToTelegram(env, pred): Promise<void>` - Sends result card photo with result caption. Uses the result card URL (card_image_url which was updated to result card by Plan 01).

       `sendDailyRecap(env): Promise<void>` - Queries today's settled predictions, computes record (e.g., "3W-1L"), running streak from recent results, and simple ROI. Formats bilingual recap message. Sends as text message (no photo). Only sends if there are results today.

       `publishDuePredictions(env): Promise<void>` - Queries predictions where:
         - published_telegram = false
         - published_site = true
         - card_image_url IS NOT NULL
         - match_date is between 2 and 3 hours from now
       For each, calls sendPredictionToTelegram. Add 1s delay between sends to avoid rate limits.

    2. Update `wrangler.toml` cron triggers to add hourly drip schedule:
       ```toml
       crons = ["0 6 * * *", "*/30 * * * *", "0 * * * *"]
       ```
       The "0 * * * *" hourly cron handles drip publishing and end-of-day recap.

    3. In `index.ts`, import publishDuePredictions and sendDailyRecap from `./telegram`.

    4. Add a new cron branch in the scheduled handler for the hourly cron `"0 * * * *"`:
       - Call `publishDuePredictions(env)` to drip-publish due predictions
       - Check if current hour (in PHT) is 23:00 (end of day). If so, call `sendDailyRecap(env)` for the daily summary
       - Wrap both in try/catch so they don't affect other crons

    5. In `resolver.ts`, import `sendResultToTelegram` from `./telegram`.

    6. In both `resolveFinishedMatches()` and `resolveNbaMatches()`, after each successful settlement AND result card generation (from Plan 01), add:
       - Call `sendResultToTelegram(env, predWithResultData)` wrapped in try/catch
       - The pred object needs: slug, card_image_url, homeTeam name, awayTeam name, homeScore, awayScore, result, pick_label_en, pick_label_tl
       - Some of this data is already available in scope from Plan 01's resolver changes (team names from join query, scores from API result)

    IMPORTANT: All Telegram calls must be wrapped in try/catch. Telegram is a distribution channel -- its failure must NEVER block prediction generation or resolution.
